The game involves having a list of $n$ five letter English words and the computer secretly picking one of these words. The player then guesses words from the list as often as they wish, but the fewer guesses the better.

After each guess the computer gives some hints by colouring the letters of the guess word, the colours signifying how close the word is to the secret word. The computer colours the letters in the following way:

  1. Letters that match the corresponding letters in the secret word are coloured green, denoted by O.

  2. Letters that are in the secret word but are not in the correct location are coloured yellow, denoted by /. If a letter appears more often in the guess than in the secret word the computer only colours the left-most letters, colouring as many letters as there are occurrences in the secret word.

  3. The rest of the letters are coloured gray, denoted by X.

As an example, if a player guesses the word error but the secret word is racer, the computer will colour it as //XXO.

Interactivity

This is an interactive problem. Your solution will be tested against an interactive judge which reads the output of your solution and prints the input your solution recieves. This interaction follows certain rules:

The judge starts by printing one line containing the integer $n$ ($1 \leq n \leq 500$), the number of words in the list. Next the judge prints $n$ lines containing the words from the list, each consisting of five English lower case letters. The words are all unique. The judge then chooses one of these words at random.

Next your solution guesses a word from the list by printing it on a single line. The judge replies by printing a single line describing the colouring as described above. If the colouring is OOOOO then your solution has guessed correctly and should exit. Otherwise your solution should guess again as described above.

Make sure to flush after each guess, for example using

  • print(..., flush=True) in Python,

  • cout << ... << endl; in C++,

  • System.out.flush(); in Java.

The problem comes with a testing tool to help test your solution.

Scoring

Your solution will be run on $100$ test cases. If your solution needs $g$ guesses to solve a test case containing $n$ words, your solution gets $\mathrm{max}(0, 1 - (g-1)^2/n)$ points for that test case. The final number of points is the sum of points received in all test cases.

It is guaranteed that $30$ test cases have $n \leq 10$, $50$ test cases have $10 < n \leq 100$ and $20$ test cases have $n = 500$.
